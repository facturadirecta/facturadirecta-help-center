<p class="no-margin">El recurso <code>products</code> te permite gestionar el catálogo de productos y servicios de tu empresa. Cada producto puede estar configurado para venta, para compra, o para ambos usos.</p><p class="no-margin"></p>
<p class="no-margin">Este artículo incluye ejemplos prácticos para los casos más comunes: productos de venta, productos de compra y productos que se usan tanto en ventas como en compras.</p><p class="no-margin"></p>
<p class="no-margin">💡 <b>Recomendación</b>: Antes de empezar a desarrollar, te recomendamos <a href="https://help.facturadirecta.com/es/articles/12608263-donde-puedo-crear-un-entorno-de-pruebas-sandbox-para-la-api" target="_blank" class="intercom-content-link">crear un entorno de pruebas (sandbox)</a> para hacer tus primeras llamadas sin afectar tus datos reales.</p><p class="no-margin"></p>
<h1 id="h_endpoints">Endpoints disponibles</h1><div class="intercom-interblocks-table-container"><table role="presentation"><tbody><tr><td style="width: 120px;"><p class="no-margin"><b>Método</b></p></td><td><p class="no-margin"><b>Endpoint</b></p></td><td><p class="no-margin"><b>Descripción</b></p></td></tr><tr><td style="width: 120px;"><p class="no-margin">GET</p></td><td><p class="no-margin">/{companyId}/products</p></td><td><p class="no-margin">Lista de productos (con filtros y paginación)</p></td></tr><tr><td style="width: 120px;"><p class="no-margin">POST</p></td><td><p class="no-margin">/{companyId}/products</p></td><td><p class="no-margin">Crear un producto</p></td></tr><tr><td style="width: 120px;"><p class="no-margin">GET</p></td><td><p class="no-margin">/{companyId}/products/{id}</p></td><td><p class="no-margin">Obtener un producto por ID</p></td></tr><tr><td style="width: 120px;"><p class="no-margin">PUT</p></td><td><p class="no-margin">/{companyId}/products/{id}</p></td><td><p class="no-margin">Actualizar un producto</p></td></tr><tr><td style="width: 120px;"><p class="no-margin">DELETE</p></td><td><p class="no-margin">/{companyId}/products/{id}</p></td><td><p class="no-margin">Eliminar un producto</p></td></tr></tbody></table></div><h1 id="h_ejemplos">Ejemplos de código</h1><p class="no-margin">Los siguientes ejemplos muestran cómo crear productos para diferentes casos de uso. Todos los ejemplos usan el método POST a <code>/{companyId}/products</code>.</p><p class="no-margin"></p>
<h2 id="h_ejemplo_sales">Producto de venta</h2><p class="no-margin">El caso más común: un producto o servicio que vendes a tus clientes.</p><p class="no-margin"></p>
<pre><code>curl -X POST \  -H "facturadirecta-api-key: TU_CLAVE_API" \  -H "Content-Type: application/json" \  -d '{    "content": {      "type": "product",      "main": {        "sku": "PV001",        "name": "Producto 001",        "currency": "EUR",        "sales": {          "price": 125,          "description": "Descripción del producto 001 que aparecerá en los documentos cuando se seleccione",          "tax": ["S_IVA_21"],          "account": "700000"        }      }    }  }' \  https://app.facturadirecta.com/api/TU_COMPANY_ID/products</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Resultado:</b> Un producto de venta con precio 125€ + IVA 21%, contabilizado en la cuenta 700000.</p><p class="no-margin"></p>
<h2 id="h_ejemplo_purchases">Producto de compra </h2><p class="no-margin">Para gastos recurrentes que quieres tener catalogados (ej: suministros, servicios externos).</p><p class="no-margin"></p>
<pre><code>{  "content": {    "type": "product",    "main": {      "sku": "PC001",      "name": "Producto comprado 001",      "currency": "EUR",      "purchases": {        "price": 125,        "description": "Descripción del producto de compra 001 que aparecerá en los documentos cuando se seleccione",        "tax": ["P_IVA_21_BC"],        "account": "600000"      }    }  }}</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Nota:</b> Los impuestos de compra usan el prefijo <code>P_</code> (ej: <code>P_IVA_21_BC</code>) mientras que los de venta usan <code>S_</code> (ej: <code>S_IVA_21</code>).</p><p class="no-margin"></p>
<h2 id="h_ejemplo_both">Producto de venta y compra</h2><p class="no-margin">Útil para productos que compras y revendes, con diferentes precios y descripciones según el contexto.</p><p class="no-margin"></p>
<pre><code>{  "content": {    "type": "product",    "main": {      "sku": "PCV009",      "name": "Producto 009",      "currency": "EUR",      "sales": {        "price": 125,        "description": "Descripción del producto en los documentos de venta",        "tax": ["S_IVA_21"],        "account": "700000"      },      "purchases": {        "price": 75,        "description": "Descripción del producto en los documentos de compra",        "tax": ["P_IVA_21_BC"],        "account": "600000"      }    }  }}</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Resultado:</b></p><ul><li><p class="no-margin">En facturas de venta: 125€ + IVA, cuenta 700000</p></li><li><p class="no-margin">En facturas de compra: 75€ + IVA, cuenta 600000</p></li><li><p class="no-margin">Margen bruto: 50€ por unidad </p></li></ul><h1 id="h_campos">Campos importantes</h1><p class="no-margin">Estos son los campos más relevantes al crear o actualizar productos:</p><p class="no-margin"></p>

<h1 id="h_uso">Uso de productos en documentos</h1><p class="no-margin">Cuando creas una factura, puedes referenciar un producto en el campo <code>document</code> de cada línea:</p><p class="no-margin"></p>
<pre><code>{  "content": {    "type": "invoice",    "main": {      "contact": "con_xxxxx",      "currency": "EUR",      "lines": [        {          "document": "pro_xxxxx",          "quantity": 2,          "unitPrice": 125,          "tax": ["S_IVA_21"],          "text": "Producto 001"        }      ]    }  }}</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Ventajas de referenciar productos:</b></p><ul><li><p class="no-margin">Los documentos quedan vinculados para análisis y reporting</p></li><li><p class="no-margin">Puedes obtener estadísticas de ventas por producto</p></li><li><p class="no-margin">Facilita la creación de líneas con datos predefinidos </p></li></ul><h1 id="h_errores">Errores comunes</h1><h2 id="h_error_name">Error: "name is required"</h2><p class="no-margin">El campo <code>name</code> es obligatorio. Indica un nombre descriptivo para el producto.</p><p class="no-margin"></p>
<h2 id="h_error_sku">Error: "sku must be unique"</h2><p class="no-margin">Ya existe otro producto con el mismo SKU. Usa un código de referencia diferente.</p><p class="no-margin"></p>
<h2 id="h_error_sales_purchases">Error: "sales or purchases required"</h2><p class="no-margin">Un producto debe tener al menos una configuración de venta (<code>sales</code>) o de compra (<code>purchases</code>).</p><p class="no-margin"></p>
<h2 id="h_error_tax">Error: "Invalid tax code"</h2><p class="no-margin">El código de impuesto no existe. Verifica que usas el prefijo correcto:</p><ul><li><p class="no-margin"><code>S_</code> para impuestos de venta</p></li><li><p class="no-margin"><code>P_</code> para impuestos de compra</p></li></ul><p class="no-margin"></p>
