<p class="no-margin">OAuth2 es el método de autenticación recomendado cuando tu aplicación necesita acceder a múltiples empresas de FacturaDirecta. Permite que los usuarios autoricen tu aplicación sin compartir sus credenciales.</p><p class="no-margin"></p>
<h1 id="h_cuando_usar">Cuándo usar OAuth2</h1><p class="no-margin">Usa OAuth2 en estos casos:</p><ul><li><p class="no-margin"><b>Aplicaciones multi-tenant</b>: Tu app permite que múltiples clientes conecten sus cuentas de FacturaDirecta</p></li><li><p class="no-margin"><b>Integraciones de terceros</b>: Ofreces una integración que los usuarios de FacturaDirecta pueden autorizar</p></li><li><p class="no-margin"><b>Aplicaciones con login de usuario</b>: Necesitas que el usuario se autentique con su cuenta de FacturaDirecta</p></li></ul><p class="no-margin">Si solo necesitas conectar <b>tu propia empresa</b>, es más sencillo usar una <a href="https://help.facturadirecta.com/es/articles/13538824-como-crear-claves-de-api" target="_blank" class="intercom-content-link">clave de API</a>.</p><p class="no-margin"></p>
<h1 id="h_diferencia_api_key">Diferencia con API Key </h1><div class="intercom-interblocks-table-container"><table role="presentation"><tbody><tr><td><p class="no-margin">Característica</p></td><td><p class="no-margin">API Key</p></td><td><p class="no-margin">OAuth2</p></td></tr><tr><td><p class="no-margin">Acceso a empresas</p></td><td><p class="no-margin">Solo una</p></td><td><p class="no-margin">Múltiples</p></td></tr><tr><td><p class="no-margin">Permisos</p></td><td><p class="no-margin">Definidos en la clave</p></td><td><p class="no-margin">Definidos por scopes + permisos del usuario</p></td></tr><tr><td><p class="no-margin">Autenticación de usuario</p></td><td><p class="no-margin">No requerida</p></td><td><p class="no-margin">Requerida</p></td></tr><tr><td><p class="no-margin">Revocación</p></td><td><p class="no-margin">Eliminando la clave</p></td><td><p class="no-margin">El usuario puede revocar desde su cuenta</p></td></tr><tr><td><p class="no-margin">Caso de uso</p></td><td><p class="no-margin">Integraciones propias</p></td><td><p class="no-margin">Aplicaciones de terceros</p></td></tr></tbody></table></div><h1 id="h_flujo_oauth">Flujo Authorization Code</h1><p class="no-margin">FacturaDirecta implementa el flujo OAuth2 Authorization Code. Este es el proceso:</p><p class="no-margin"></p>
<h2 id="h_paso_1">Paso 1: Redirigir al usuario</h2><p class="no-margin">Envía al usuario a la página de autorización:</p><p class="no-margin"></p>
<pre><code>https://app.facturadirecta.com/oauth/authorize  ?client_id=facturadirecta-api  &amp;response_type=code  &amp;redirect_uri=https://tu-app.com/callback  &amp;scope=invoices:read+invoices:write+contacts:read</code></pre><p class="no-margin"></p>
<p class="no-margin">Parámetros:</p><ul><li><p class="no-margin"><b>client_id</b>: <code>facturadirecta-api</code> (valor fijo)</p></li><li><p class="no-margin"><b>response_type</b>: <code>code</code></p></li><li><p class="no-margin"><b>redirect_uri</b>: URL de tu aplicación donde se enviará el código</p></li><li><p class="no-margin"><b>scope</b>: Permisos que solicitas (separados por <code>+</code> o espacio)</p></li></ul><p class="no-margin"></p>
<h2 id="h_paso_2">Paso 2: El usuario autoriza</h2><p class="no-margin">El usuario inicia sesión en FacturaDirecta (si no lo ha hecho) y ve una pantalla con los permisos que solicitas. Si acepta, FacturaDirecta redirige a tu <code>redirect_uri</code> con un código:</p><p class="no-margin"></p>
<pre><code>https://tu-app.com/callback?code=abc123xyz</code></pre><p class="no-margin"></p>
<h2 id="h_paso_3">Paso 3: Intercambiar código por token</h2><p class="no-margin">Tu servidor intercambia el código por un token de acceso:</p><p class="no-margin"></p>
<pre><code>curl -X POST \     -H "Content-Type: application/x-www-form-urlencoded" \     -d "grant_type=authorization_code" \     -d "code=abc123xyz" \     -d "client_id=facturadirecta-api" \     -d "redirect_uri=https://tu-app.com/callback" \     https://app.facturadirecta.com/oauth/token</code></pre><p class="no-margin"></p>
<p class="no-margin">Respuesta:</p><pre><code>{  "access_token": "eyJhbGciOiJSUzI1NiIs...",  "token_type": "Bearer",  "expires_in": 3600,  "refresh_token": "dGhpcyBpcyBhIHJlZnJlc2..."}</code></pre><p class="no-margin"></p>
<h2 id="h_paso_4">Paso 4: Usar el token</h2><p class="no-margin">Incluye el token en tus llamadas a la API:</p><p class="no-margin"></p>
<pre><code>curl -H "Authorization: Bearer eyJhbGciOiJSUzI1NiIs..." \     https://app.facturadirecta.com/api/{companyId}/invoices</code></pre><p class="no-margin"></p>
<h1 id="h_obtener_empresas">Obtener las empresas del usuario</h1><p class="no-margin">Después de autenticar, puedes consultar a qué empresas tiene acceso el usuario:</p><p class="no-margin"></p>
<pre><code>curl -H "Authorization: Bearer {token}" \     https://app.facturadirecta.com/api/profile</code></pre><p class="no-margin"></p>
<p class="no-margin">La respuesta incluye la lista de empresas accesibles con sus IDs.</p><p class="no-margin"></p>
<h1 id="h_renovar_token">Renovar el token</h1><p class="no-margin">Cuando el token expire, usa el refresh token para obtener uno nuevo:</p><p class="no-margin"></p>
<pre><code>curl -X POST \     -H "Content-Type: application/x-www-form-urlencoded" \     -d "grant_type=refresh_token" \     -d "refresh_token=dGhpcyBpcyBhIHJlZnJlc2..." \     -d "client_id=facturadirecta-api" \     https://app.facturadirecta.com/oauth/token</code></pre><p class="no-margin"></p>
<h1 id="h_scopes">Scopes disponibles</h1><p class="no-margin">Los scopes determinan qué permisos solicitas. Son los mismos que los de las claves de API:</p><ul><li><p class="no-margin"><code>contacts:read</code>, <code>contacts:write</code></p></li><li><p class="no-margin"><code>products:read</code>, <code>products:write</code></p></li><li><p class="no-margin"><code>invoices:read</code>, <code>invoices:write</code></p></li><li><p class="no-margin"><code>estimates:read</code>, <code>estimates:write</code></p></li><li><p class="no-margin"><code>deliveryNotes:read</code>, <code>deliveryNotes:write</code></p></li><li><p class="no-margin"><code>bills:read</code>, <code>bills:write</code></p></li><li><p class="no-margin"><code>payrolls:read</code>, <code>payrolls:write</code></p></li><li><p class="no-margin"><code>banks:read</code>, <code>banks:write</code></p></li><li><p class="no-margin"><code>accounting:read</code>, <code>accounting:write</code></p></li><li><p class="no-margin"><code>settings:read</code>, <code>settings:write</code></p></li></ul><p class="no-margin"></p>
<p class="no-margin"><b>Nota</b>: El acceso efectivo es la intersección entre los scopes solicitados y los permisos que el usuario tiene en la empresa.</p><p class="no-margin"></p>
<p class="no-margin">💡 <b>Nota</b>: OAuth2 también funciona con los <a href="https://help.facturadirecta.com/es/articles/12608263-donde-puedo-crear-un-entorno-de-pruebas-sandbox-para-la-api" target="_blank" class="intercom-content-link">entornos de prueba (sandbox)</a>. Los usuarios pueden autorizar tu aplicación en sus sandboxes para que pruebes la integración.</p><p class="no-margin"></p>
