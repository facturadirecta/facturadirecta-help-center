<p class="no-margin">⚠️ <b>AVISO</b>: Los webhooks están disponibles en los planes Avanzado y Total.</p><p class="no-margin"></p>
<p class="no-margin">Cuando FacturaDirecta envía un evento a tu endpoint, incluye una firma digital en las cabeceras HTTP. Verificar esta firma te permite confirmar que la notificación proviene realmente de FacturaDirecta y que no ha sido manipulada por un tercero.</p><p class="no-margin"></p>
<p class="no-margin">💡 <b>Recomendación</b>: Verifica siempre la firma antes de procesar un evento. Si la firma no coincide, descarta la petición.</p><p class="no-margin"></p>
<h1 id="h_como_funciona">Cómo funciona la firma</h1><p class="no-margin">Cada petición webhook incluye dos cabeceras HTTP:</p><div class="intercom-interblocks-table-container"><table role="presentation"><tbody><tr><td><p class="no-margin"><b>Cabecera</b></p></td><td><p class="no-margin"><b>Formato</b></p></td><td><p class="no-margin"><b>Descripción</b></p></td></tr><tr><td><p class="no-margin"><code>Webhook-Signature</code></p></td><td><p class="no-margin"><code>t=&lt;timestamp&gt;,v1=&lt;hmac_hex&gt;</code></p></td><td><p class="no-margin">Timestamp del envío y firma HMAC SHA256</p></td></tr><tr><td><p class="no-margin"><code>Webhook-Timestamp</code></p></td><td><p class="no-margin"><code>&lt;unix_timestamp&gt;</code></p></td><td><p class="no-margin">Timestamp del envío (en segundos)</p></td></tr></tbody></table></div><p class="no-margin">La firma se calcula sobre una combinación del timestamp y el body de la petición, usando tu signing secret como clave.</p><p class="no-margin"></p>
<h1 id="h_paso_a_paso">Verificación paso a paso</h1><p class="no-margin">Para verificar la firma de un evento:</p><ol><li><p class="no-margin"><b>Extrae el timestamp y la firma</b> de la cabecera <code>Webhook-Signature</code>. El formato es <code>t=&lt;timestamp&gt;,v1=&lt;firma&gt;</code></p></li><li><p class="no-margin"><b>Construye el string a firmar</b> concatenando el timestamp, un punto (<code>.</code>) y el body completo de la petición: <code>&lt;timestamp&gt;.&lt;body&gt;</code></p></li><li><p class="no-margin"><b>Calcula el HMAC SHA256</b> del string anterior usando tu signing secret como clave</p></li><li><p class="no-margin"><b>Compara la firma calculada</b> con la firma recibida usando una comparación de tiempo constante (timing-safe)</p></li><li><p class="no-margin"><b>Verifica la ventana de tiempo</b>: comprueba que el timestamp no tiene más de 5 minutos de antigüedad para protegerte contra ataques de repetición</p></li></ol><h1 id="h_ejemplos_codigo">Ejemplos de código</h1><h2 id="h_ejemplo_nodejs">Node.js</h2><pre><code>const crypto = require('crypto');function verifyWebhookSignature(body, signatureHeader, secret) {  // 1. Extraer timestamp y firma  const parts = {};  signatureHeader.split(',').forEach(part =&gt; {    const [key, value] = part.split('=');    parts[key] = value;  });  const timestamp = parts['t'];  const signature = parts['v1'];  // 2. Construir el string a firmar  const signedPayload = `${timestamp}.${body}`;  // 3. Calcular HMAC SHA256  const expectedSignature = crypto    .createHmac('sha256', secret)    .update(signedPayload)    .digest('hex');  // 4. Comparar con timing-safe  const isValid = crypto.timingSafeEqual(    Buffer.from(signature, 'hex'),    Buffer.from(expectedSignature, 'hex')  );  // 5. Verificar ventana de tiempo (5 minutos)  const now = Math.floor(Date.now() / 1000);  const isFresh = Math.abs(now - parseInt(timestamp)) &lt; 300;  return isValid &amp;&amp; isFresh;}</code></pre><p class="no-margin"></p>
<h2 id="h_ejemplo_python">Python</h2><pre><code>import hmacimport hashlibimport timedef verify_webhook_signature(body, signature_header, secret):    # 1. Extraer timestamp y firma    parts = dict(p.split('=', 1) for p in signature_header.split(','))    timestamp = parts['t']    signature = parts['v1']    # 2. Construir el string a firmar    signed_payload = f"{timestamp}.{body}"    # 3. Calcular HMAC SHA256    expected = hmac.new(        secret.encode('utf-8'),        signed_payload.encode('utf-8'),        hashlib.sha256    ).hexdigest()    # 4. Comparar con timing-safe    is_valid = hmac.compare_digest(signature, expected)    # 5. Verificar ventana de tiempo (5 minutos)    is_fresh = abs(time.time() - int(timestamp)) &lt; 300    return is_valid and is_fresh</code></pre><p class="no-margin"></p>
<h2 id="h_ejemplo_php">PHP</h2><pre><code>function verifyWebhookSignature($body, $signatureHeader, $secret) {    // 1. Extraer timestamp y firma    $parts = [];    foreach (explode(',', $signatureHeader) as $part) {        list($key, $value) = explode('=', $part, 2);        $parts[$key] = $value;    }    $timestamp = $parts['t'];    $signature = $parts['v1'];    // 2. Construir el string a firmar    $signedPayload = "{$timestamp}.{$body}";    // 3. Calcular HMAC SHA256    $expected = hash_hmac('sha256', $signedPayload, $secret);    // 4. Comparar con timing-safe    $isValid = hash_equals($signature, $expected);    // 5. Verificar ventana de tiempo (5 minutos)    $isFresh = abs(time() - intval($timestamp)) &lt; 300;    return $isValid &amp;&amp; $isFresh;}</code></pre><p class="no-margin"></p>
<h1 id="h_proteccion_replay">Protección anti-replay</h1><p class="no-margin">La verificación del timestamp (paso 5) protege contra ataques de repetición (replay attacks). Si alguien intercepta un evento válido e intenta reenviarlo más tarde, la ventana de <b>5 minutos</b> hace que la firma se considere expirada.</p><p class="no-margin"></p>
<p class="no-margin">Comprueba siempre que el timestamp del evento no difiere en más de 300 segundos (5 minutos) respecto a la hora actual de tu servidor.</p><p class="no-margin"></p>
<h1 id="h_signing_secret">El signing secret</h1><p class="no-margin">El signing secret es la clave que se usa para calcular y verificar las firmas. Tiene el formato <code>whsec_&lt;valor_hexadecimal&gt;</code>.</p><p class="no-margin"></p>
<p class="no-margin">El signing secret solo se muestra en dos momentos:</p><ul><li><p class="no-margin">Al <b>crear un nuevo endpoint</b></p></li><li><p class="no-margin">Al <b>rotar el secret</b> de un endpoint existente</p></li></ul><p class="no-margin">En ambos casos, guárdalo en un lugar seguro (por ejemplo, en las variables de entorno de tu servidor). No podrás volver a consultarlo después.</p><p class="no-margin"></p>
<p class="no-margin">Si pierdes el signing secret, puedes rotar el secret desde el detalle del endpoint en FacturaDirecta. Al hacerlo, se genera uno nuevo y el anterior deja de funcionar inmediatamente.</p><p class="no-margin"></p>
<h1 id="h_buenas_practicas">Buenas prácticas</h1><ul><li><p class="no-margin"><b>Verifica siempre la firma</b>: nunca proceses un evento sin antes comprobar que la firma es válida</p></li><li><p class="no-margin"><b>Usa comparación timing-safe</b>: evita comparar strings directamente (<code>===</code>). Usa las funciones de comparación de tiempo constante de tu lenguaje (<code>timingSafeEqual</code>, <code>compare_digest</code>, <code>hash_equals</code>)</p></li><li><p class="no-margin"><b>Responde rápido con HTTP 200</b>: tu servidor debe responder con un código 2xx lo antes posible. Si necesitas hacer procesamiento largo, acepta el evento primero y procésalo de forma asíncrona</p></li><li><p class="no-margin"><b>Responde HTTP 410 para desuscribirte</b>: si ya no quieres recibir eventos en un endpoint, responde con 410 Gone y FacturaDirecta lo desactivará automáticamente</p></li></ul>
