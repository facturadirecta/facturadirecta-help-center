<h1 id="api-keys">API keys</h1><p class="no-margin"></p>
<p class="no-margin">Una API key es una credencial <b>por empresa</b> para autenticar integraciones de máquina a máquina. Es el método alternativo a OAuth2 cuando tu integración solo necesita acceder a una empresa concreta y no necesita actuar en nombre de un usuario humano. Ver <a href="https://help.facturadirecta.com/es/articles/13538825" target="_blank" class="intercom-content-link">Autenticación</a> para el contexto general.</p><p class="no-margin"></p>
<p class="no-margin">Este recurso permite gestionar las API keys vía la propia API: crear, listar, consultar y borrar.</p><p class="no-margin"></p>
<p class="no-margin">En los ejemplos de esta página, los IDs y prefijos son ilustrativos. Los reales se devuelven al crear cada API key.</p><p class="no-margin"></p>
<h2 id="cuatro-principios-duros-del-recurso">Cuatro principios duros del recurso</h2><p class="no-margin"></p>
<p class="no-margin">Estas reglas son distintas a las de cualquier otro recurso y conviene tenerlas claras antes de operar:</p><p class="no-margin"></p>
<ol><li><p class="no-margin"><b>La clave completa solo se ve una vez</b>, en la respuesta de la creación. Después FacturaDirecta solo guarda un hash y el prefijo para identificación. Si la pierdes, hay que crear otra.</p></li><li><p class="no-margin"><b>No hay endpoint de modificación</b> (<code>PUT</code>). Las API keys no se actualizan: para cambiar nombre o scopes hay que crear una nueva y borrar la antigua. Esto evita reasignar scopes a una clave en uso por descuido.</p></li><li><p class="no-margin"><b>No se pueden borrar API keys creadas manualmente</b> desde la interfaz (<code>managedViaApi: false</code>). Solo las que tu integración ha creado por API (<code>managedViaApi: true</code>) son borrables por API. Es una protección contra que un cliente automático elimine accidentalmente las credenciales que un humano configuró.</p></li><li><p class="no-margin"><b>Una API key no puede borrarse a sí misma.</b> Si autenticas con la API key <code>K</code> e intentas <code>DELETE /apiKeys/K</code>, la API rechaza con <code>403</code>.</p></li></ol><p class="no-margin"></p>
<h2 id="formato-de-la-clave">Formato de la clave</h2><p class="no-margin"></p>
<p class="no-margin">Al crear una API key, la respuesta incluye un campo <code>apiKey</code> con la clave <b>completa</b>, en formato:</p><p class="no-margin"></p>
<pre><code>&lt;prefijo&gt;.&lt;sufijo&gt;</code></pre><p class="no-margin"></p>
<ul><li><p class="no-margin"><b>Prefijo</b>: 6 caracteres alfanuméricos. Identifica la clave en logs, listas, y queda asociado a las llamadas que realiza. Es lo que ves después en el campo <code>prefix</code> de las respuestas.</p></li><li><p class="no-margin"><b>Sufijo</b>: 32 caracteres alfanuméricos. Es el secreto real. Junto con el prefijo forman la clave única.</p></li></ul><p class="no-margin"></p>
<p class="no-margin">Ejemplo de clave válida:</p><p class="no-margin"></p>
<pre><code>aB3cD9.xY7zN2qK4mP8rT5sJ1hF6gL0vW3uE</code></pre><p class="no-margin"></p>
<p class="no-margin">Para usarla en peticiones a la API, se envía en el header <code>facturadirecta-api-key</code> (sin <code>Bearer</code>, sin esquema). Ver <a href="https://help.facturadirecta.com/es/articles/13538825#api-key" target="_blank" class="intercom-content-link">Autenticación</a>.</p><p class="no-margin"></p>
<h2 id="estructura">Estructura</h2><p class="no-margin"></p>
<p class="no-margin"><code>PublicApiKeyInfo</code> (lo que devuelven los GETs):</p><p class="no-margin"></p>
<div class="intercom-interblocks-table-container"><table role="presentation"><tbody><tr><td><p class="no-margin">Campo</p></td><td><p class="no-margin">Tipo</p></td><td><p class="no-margin">Obligatorio</p></td><td><p class="no-margin">Significado</p></td></tr><tr><td><p class="no-margin"><code>id</code></p></td><td><p class="no-margin">string</p></td><td><p class="no-margin">sí</p></td><td><p class="no-margin">Identificador único de la API key (UUID v4 sin prefijo). Se usa en los paths como <code>{id}</code>.</p></td></tr><tr><td><p class="no-margin"><code>name</code></p></td><td><p class="no-margin">string</p></td><td><p class="no-margin">sí</p></td><td><p class="no-margin">Nombre descriptivo elegido al crear.</p></td></tr><tr><td><p class="no-margin"><code>prefix</code></p></td><td><p class="no-margin">string</p></td><td><p class="no-margin">sí</p></td><td><p class="no-margin">Prefijo de 6 caracteres.</p></td></tr><tr><td><p class="no-margin"><code>scopes</code></p></td><td><p class="no-margin">string[]</p></td><td><p class="no-margin">sí</p></td><td><p class="no-margin">Scopes autorizados de la clave.</p></td></tr><tr><td><p class="no-margin"><code>creationTime</code></p></td><td><p class="no-margin">string ISO 8601</p></td><td><p class="no-margin">sí</p></td><td><p class="no-margin">Fecha de creación.</p></td></tr><tr><td><p class="no-margin"><code>createdBy</code></p></td><td><p class="no-margin">string</p></td><td><p class="no-margin">no</p></td><td><p class="no-margin">Usuario que creó la clave.</p></td></tr><tr><td><p class="no-margin"><code>managedViaApi</code></p></td><td><p class="no-margin">boolean</p></td><td><p class="no-margin">sí</p></td><td><p class="no-margin"><code>true</code> si la clave fue creada vía API y por tanto es borrable vía API. <code>false</code> si se creó desde la interfaz.</p></td></tr></tbody></table></div><p class="no-margin"></p>
<p class="no-margin">Al crear, además del <code>content: PublicApiKeyInfo</code>, la respuesta incluye el campo <code>apiKey</code> con la clave completa.</p><p class="no-margin"></p>
<h2 id="operaciones">Operaciones</h2><p class="no-margin"></p>
<ul><li><p class="no-margin"><a href="#lista-de-api-keys" target="_blank" class="intercom-content-link">Lista de API keys</a></p></li><li><p class="no-margin"><a href="#crear-api-key" target="_blank" class="intercom-content-link">Crear API key</a></p></li><li><p class="no-margin"><a href="#obtener-una-api-key" target="_blank" class="intercom-content-link">Obtener una API key</a></p></li><li><p class="no-margin"><a href="#borrar-una-api-key" target="_blank" class="intercom-content-link">Borrar una API key</a></p></li></ul><p class="no-margin"></p>
<h2 id="lista-de-api-keys">Lista de API keys</h2><p class="no-margin"></p>
<p class="no-margin"><code>GET /{companyId}/apiKeys</code> devuelve todas las API keys configuradas en la empresa, sin datos sensibles (solo prefijos, no claves completas).</p><p class="no-margin"></p>
<p class="no-margin"><b>Parámetros de consulta específicos:</b> ninguno.</p><p class="no-margin"></p>
<p class="no-margin"><b>Parámetros globales aceptados:</b> <code>accept-version</code>.</p><p class="no-margin"></p>
<p class="no-margin"><b>Notas:</b></p><p class="no-margin"></p>
<ul><li><p class="no-margin">La lista incluye <b>todas</b> las API keys de la empresa, las creadas vía API y las creadas manualmente. Distínguelas por <code>managedViaApi</code>.</p></li><li><p class="no-margin">Sin paginación: devuelve el conjunto completo (ver patrón "sin paginación" en la <a href="https://help.facturadirecta.com/es/articles/15084464" target="_blank" class="intercom-content-link">guía de paginación</a>).</p></li></ul><p class="no-margin"></p>
<h3 id="copy-as-curl">Copy as cURL</h3><p class="no-margin"></p>
<pre><code>curl -s -H "Authorization: Bearer $ACCESS_TOKEN" \  "https://app.facturadirecta.com/api/$COMPANY_ID/apiKeys"</code></pre><p class="no-margin"></p>
<h2 id="crear-api-key">Crear API key</h2><p class="no-margin"></p>
<p class="no-margin"><code>POST /{companyId}/apiKeys</code> crea una API key.</p><p class="no-margin"></p>
<p class="no-margin"><b>Parámetros del body:</b></p><p class="no-margin"></p>
<ul><li><p class="no-margin"><code>name</code> (obligatorio) — nombre descriptivo para identificarla en la lista.</p></li><li><p class="no-margin"><code>scopes</code> (obligatorio, mínimo 1) — lista de scopes que la nueva clave podrá usar. <b>Si el caller es una API key, los scopes deben ser subconjunto de los suyos</b>. Si el caller es OAuth, puede asignar cualquier scope disponible.</p></li></ul><p class="no-margin"></p>
<p class="no-margin"><b>Respuesta:</b></p><p class="no-margin"></p>
<pre><code>{  "content": { "id": "...", "name": "...", "prefix": "...", "scopes": [...], ... },  "apiKey": "&lt;prefijo&gt;.&lt;sufijo&gt;"}</code></pre><p class="no-margin"></p>
<p class="no-margin">El campo <code>apiKey</code> <b>solo se ve aquí</b>. Guárdalo en seguro antes de seguir; no hay forma de recuperarlo después.</p><p class="no-margin"></p>
<p class="no-margin"><b>Parámetros globales aceptados:</b> <code>accept-version</code>.</p><p class="no-margin"></p>
<h3 id="ejemplo-de-request-json">Ejemplo de request JSON</h3><p class="no-margin"></p>
<pre><code>{  "name": "Mi API key",  "scopes": ["contacts:read"]}</code></pre><p class="no-margin"></p>
<h3 id="copy-as-curl">Copy as cURL</h3><p class="no-margin"></p>
<pre><code>curl -s -H "Authorization: Bearer $ACCESS_TOKEN" -H "Content-Type: application/json" \  -d '{"name":"Mi API key","scopes":["contacts:read"]}' \  "https://app.facturadirecta.com/api/$COMPANY_ID/apiKeys"</code></pre><p class="no-margin"></p>
<h2 id="obtener-una-api-key">Obtener una API key</h2><p class="no-margin"></p>
<p class="no-margin"><code>GET /{companyId}/apiKeys/{id}</code> devuelve los metadatos de una API key por su ID.</p><p class="no-margin"></p>
<p class="no-margin">No incluye la clave completa (solo el prefijo). Esa información ya no existe en FacturaDirecta tras la creación.</p><p class="no-margin"></p>
<p class="no-margin"><b>Parámetros globales aceptados:</b> <code>accept-version</code>.</p><p class="no-margin"></p>
<h3 id="copy-as-curl">Copy as cURL</h3><p class="no-margin"></p>
<pre><code>curl -s -H "Authorization: Bearer $ACCESS_TOKEN" \  "https://app.facturadirecta.com/api/$COMPANY_ID/apiKeys/3f8b2c14-7d4e-4a9c-b6f1-2a5e8d3c4b7f"</code></pre><p class="no-margin"></p>
<h2 id="borrar-una-api-key">Borrar una API key</h2><p class="no-margin"></p>
<p class="no-margin"><code>DELETE /{companyId}/apiKeys/{id}</code> elimina la API key.</p><p class="no-margin"></p>
<p class="no-margin"><b>Restricciones:</b></p><p class="no-margin"></p>
<ul><li><p class="no-margin">La API key objetivo debe tener <code>managedViaApi: true</code>. Si fue creada desde la interfaz, devuelve <code>403 Forbidden</code> con el mensaje "No se puede eliminar una API key que no fue creada vía API. Las API keys creadas manualmente solo pueden eliminarse desde la interfaz de usuario."</p></li><li><p class="no-margin">Una API key no puede borrarse a sí misma. Si autenticas con la API key <code>K</code> e intentas borrarla, devuelve <code>403 Forbidden</code> con el mensaje "Una API key no puede eliminarse a sí misma".</p></li><li><p class="no-margin">Si el <code>id</code> no corresponde a ninguna API key existente, devuelve <code>403 Forbidden</code> (no <code>404</code>).</p></li></ul><p class="no-margin"></p>
<p class="no-margin"><b>Respuesta:</b> <code>{ result: true }</code>.</p><p class="no-margin"></p>
<p class="no-margin"><b>Parámetros globales aceptados:</b> <code>accept-version</code>.</p><p class="no-margin"></p>
<h3 id="copy-as-curl">Copy as cURL</h3><p class="no-margin"></p>
<pre><code>curl -s -H "Authorization: Bearer $ACCESS_TOKEN" -X DELETE \  "https://app.facturadirecta.com/api/$COMPANY_ID/apiKeys/3f8b2c14-7d4e-4a9c-b6f1-2a5e8d3c4b7f"</code></pre><p class="no-margin"></p>
<h2 id="patrn-de-rotacin">Patrón de rotación</h2><p class="no-margin"></p>
<p class="no-margin">Para rotar una API key sin interrumpir el servicio:</p><p class="no-margin"></p>
<ol><li><p class="no-margin">Crea una <b>nueva</b> API key con los mismos scopes que la antigua.</p></li><li><p class="no-margin">Actualiza tu integración para usar la nueva clave.</p></li><li><p class="no-margin">Verifica que la integración funciona con la nueva clave (deja un margen razonable).</p></li><li><p class="no-margin">Borra la antigua con <code>DELETE /apiKeys/{id}</code> (solo si fue creada vía API).</p></li></ol><p class="no-margin"></p>
<p class="no-margin">No hay "rotateSecret" como en webhooks: la API key es inmutable. La rotación se hace siempre creando nueva y borrando antigua.</p><p class="no-margin"></p>
<h2 id="recomendaciones">Recomendaciones</h2><p class="no-margin"></p>
<ul><li><p class="no-margin"><b>Pide los scopes mínimos.</b> Si tu integración solo lee contactos, no pidas <code>contacts:write</code>. Es más seguro y más fácil de auditar.</p></li><li><p class="no-margin"><b>Una API key por integración, no compartas.</b> Si una integración deja de usarse, borra su API key.</p></li><li><p class="no-margin"><b>Guarda la clave (<code>&lt;prefijo&gt;.&lt;sufijo&gt;</code>) en un gestor de secretos</b>, no en código fuente ni en logs.</p></li><li><p class="no-margin"><b>Las API keys son por empresa.</b> Para multi-tenant SaaS, OAuth2 con <code>offline_access</code> es el patrón correcto (no API keys una por empresa).</p></li><li><p class="no-margin"><b>Para auto-aprovisionamiento de claves desde tu sistema</b>, usa OAuth2 para autenticar tu sistema admin y crear API keys para cada cliente. Sigue siendo más seguro que distribuir API keys hardcodeadas.</p></li></ul><p class="no-margin"></p>
<h2 id="errores-comunes">Errores comunes</h2><p class="no-margin"></p>
<ul><li><p class="no-margin"><code>400 ValidationError</code> — <code>scopes</code> vacío o <code>name</code> ausente.</p></li><li><p class="no-margin"><code>403 Forbidden</code> — el <code>id</code> no existe, o <code>managedViaApi: false</code>, o intento de auto-eliminación.</p></li><li><p class="no-margin"><code>403 Forbidden</code> — scope solicitado fuera del subconjunto del caller (cuando el caller autentica con API key).</p></li></ul><p class="no-margin"></p>
<p class="no-margin">Ver <a href="https://help.facturadirecta.com/es/articles/15084452" target="_blank" class="intercom-content-link">Errores y validaciones</a> para el formato general.</p><p class="no-margin"></p>

<h2 id="scopes">Scopes</h2><p class="no-margin"></p>
<ul><li><p class="no-margin"><b><code>apiKeys:read</code></b> — Lectura de API keys.</p></li><li><p class="no-margin"><b><code>apiKeys:write</code></b> — Gestión de API keys (creación y eliminación).</p></li></ul>
