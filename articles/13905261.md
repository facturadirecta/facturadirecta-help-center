<p class="no-margin">⚠️ <b>AVISO</b>: Los webhooks están disponibles en los planes Avanzado y Total.</p><p class="no-margin"></p>
<p class="no-margin">La API de webhooks te permite gestionar endpoints, consultar eventos y reenviar notificaciones de forma programática. Puedes crear endpoints, suscribirte a eventos, revisar el historial de envíos y rotar signing secrets, todo mediante la API.</p><p class="no-margin"></p>
<p class="no-margin">💡 <b>Recomendación</b>: Antes de empezar a desarrollar, te recomendamos <a href="https://help.facturadirecta.com/es/articles/12608263-donde-puedo-crear-un-entorno-de-pruebas-sandbox-para-la-api" target="_blank" class="intercom-content-link">crear un entorno de pruebas (sandbox)</a> para hacer tus primeras llamadas sin afectar tus datos reales.</p><p class="no-margin"></p>
<h1 id="h_autenticacion">Autenticación</h1><p class="no-margin">La API de webhooks acepta dos métodos de autenticación:</p><ul><li><p class="no-margin"><b>API Key</b>: añade la cabecera <code>facturadirecta-api-key: TU_CLAVE_API</code>. La clave debe tener los permisos <code>webhooks:read</code> y/o <code>webhooks:write</code></p></li><li><p class="no-margin"><b>OAuth2</b>: solicita los scopes <code>webhooks:read</code> y/o <code>webhooks:write</code> en el flujo de autorización</p></li></ul><p class="no-margin"></p>

<h1 id="h_ejemplos">Ejemplos de código</h1><h2 id="h_ejemplo_crear">Crear un endpoint</h2><pre><code>curl -X POST \  -H "facturadirecta-api-key: TU_CLAVE_API" \  -H "Content-Type: application/json" \  -d '{    "url": "https://tu-servidor.com/webhook",    "name": "Integración ERP",    "events": ["invoice.created", "invoice.updated", "contact.created"]  }' \  https://app.facturadirecta.com/api/TU_COMPANY_ID/webhooks/endpoints</code></pre><p class="no-margin"></p>
<p class="no-margin">La respuesta incluye el signing secret (solo se muestra en este momento):</p><pre><code>{  "data": {    "id": "wep_abc123",    "url": "https://tu-servidor.com/webhook",    "name": "Integración ERP",    "events": ["invoice.created", "invoice.updated", "contact.created"],    "status": "active",    "secret": "whsec_a1b2c3d4e5f6..."  }}</code></pre><p class="no-margin"></p>
<h2 id="h_ejemplo_listar">Listar endpoints</h2><pre><code>curl -H "facturadirecta-api-key: TU_CLAVE_API" \  https://app.facturadirecta.com/api/TU_COMPANY_ID/webhooks/endpoints</code></pre><p class="no-margin"></p>
<h2 id="h_ejemplo_eventos">Consultar eventos</h2><pre><code>curl -H "facturadirecta-api-key: TU_CLAVE_API" \  "https://app.facturadirecta.com/api/TU_COMPANY_ID/webhooks/events?endpoint_id=wep_abc123&amp;status=failed&amp;limit=10"</code></pre><p class="no-margin"></p>
<h2 id="h_ejemplo_reenviar">Reenviar un evento</h2><pre><code>curl -X POST \  -H "facturadirecta-api-key: TU_CLAVE_API" \  https://app.facturadirecta.com/api/TU_COMPANY_ID/webhooks/events/whe_xyz789/retry</code></pre><p class="no-margin"></p>
<h2 id="h_ejemplo_rotar">Rotar signing secret</h2><pre><code>curl -X POST \  -H "facturadirecta-api-key: TU_CLAVE_API" \  https://app.facturadirecta.com/api/TU_COMPANY_ID/webhooks/endpoints/wep_abc123/rotate-secret</code></pre><p class="no-margin"></p>
<p class="no-margin">La respuesta incluye el nuevo signing secret:</p><pre><code>{  "data": {    "id": "wep_abc123",    "secret": "whsec_nuevo_secret..."  }}</code></pre><p class="no-margin"></p>

<h1 id="h_errores">Errores comunes</h1><h2 id="h_error_403">Error 403: Forbidden</h2><p class="no-margin">Tu clave de API no tiene los permisos necesarios. Verifica que la clave incluye los scopes <code>webhooks:read</code> y/o <code>webhooks:write</code> según la operación.</p><p class="no-margin"></p>
<h2 id="h_error_422">Error 422: Unprocessable Entity</h2><p class="no-margin">Los datos enviados no son válidos. Causas habituales:</p><ul><li><p class="no-margin">La URL no empieza por <code>https://</code></p></li><li><p class="no-margin">La URL apunta a una IP privada (localhost, 127.x, 10.x, 192.168.x)</p></li><li><p class="no-margin">No se ha seleccionado ningún evento</p></li><li><p class="no-margin">El tipo de evento no existe en el catálogo</p></li></ul><p class="no-margin"></p>
<h2 id="h_error_404">Error 404: Not Found</h2><p class="no-margin">El endpoint o evento no existe. Verifica que el ID es correcto y pertenece a tu empresa.</p>
