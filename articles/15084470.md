<h1 id="perfil-de-usuario"><b>Perfil de usuario</b></h1>
<p class="no-margin"></p>
<p class="no-margin">El recurso <code>profile</code> devuelve los datos básicos del <b>usuario
autenticado</b> y la lista de <b>empresas a las que tiene acceso</b>. Es
la operación que usa toda integración OAuth para resolver qué
<code>companyId</code> puede usar en el resto de llamadas.</p>
<p class="no-margin"></p>
<h2 id="excepciones-de-este-endpoint"><b>Excepciones de este endpoint</b></h2>
<p class="no-margin"></p>
<p class="no-margin"><code>/profile</code> es el único endpoint de la API pública que rompe con dos
convenciones del resto del catálogo:</p>
<p class="no-margin"></p>
<ul>
<li><p class="no-margin"><b>No lleva <code>companyId</code> en la ruta.</b> El path es literalmente
<code>/profile</code>: el usuario puede tener acceso a varias empresas, y este
endpoint sirve precisamente para descubrirlas antes de fijar una.</p></li>
<li><p class="no-margin"><b>Es OAuth-only.</b> Las <a href="https://help.facturadirecta.com/es/articles/13538824">API Keys</a> <b>no pueden
llamar</b> a este endpoint: una API key está ya ligada a una empresa
concreta, así que no tiene sentido pedir su perfil de empresas.</p></li>
</ul>
<p class="no-margin"></p>
<h2 id="scope"><b>Scope</b></h2>
<p class="no-margin"></p>
<p class="no-margin">El scope efectivo es <code>profile</code>. No aparece como permiso
seleccionable porque <b>se incluye automáticamente</b> en el cliente
OAuth <code>facturadirecta-api</code> al hacer login. No hace falta pedirlo
explícitamente.</p>
<p class="no-margin"></p>

<h2 id="obtener-el-perfil"><b>Obtener el perfil</b></h2>
<p class="no-margin"></p>
<p class="no-margin"><code>GET /profile</code> devuelve el perfil del usuario autenticado y sus
empresas.</p>
<p class="no-margin"></p>
<p class="no-margin"><b>No lleva parámetros de path ni de consulta.</b> Solo el header
<code>accept-version</code> para fijar la versión de la API. Ver
<a href="https://help.facturadirecta.com/es/articles/13538825">Autenticación</a>.</p>
<p class="no-margin"></p>
<h6 id="copy-as-curl"><b>Copy as cURL</b></h6>
<p class="no-margin"></p>
<pre><code class="language-shell">curl -s -H &quot;Authorization: Bearer $ACCESS_TOKEN&quot; \
  &quot;https://app.facturadirecta.com/api/profile&quot;</code></pre>
<p class="no-margin"></p>
<p class="no-margin">Respuesta típica:</p>

<h2 id="flujo-tpico-de-integracin"><b>Flujo típico de integración</b></h2>
<p class="no-margin"></p>
<ol>
<li><p class="no-margin">El usuario completa el login OAuth. Tu integración recibe el
<code>access_token</code>.</p></li>
<li><p class="no-margin">Llamas a <code>GET /profile</code> para descubrir las empresas accesibles.</p></li>
<li><p class="no-margin">Eliges una empresa (UI: dejas que el usuario seleccione; o
automatización: filtras por <code>taxCode</code> o <code>id</code> conocido).</p></li>
<li><p class="no-margin">Usas su <code>id</code> como <code>{companyId}</code> en todas las llamadas siguientes
a la API.</p></li>
</ol>
<p class="no-margin"></p>
<h2 id="recomendaciones"><b>Recomendaciones</b></h2>
<p class="no-margin"></p>
<ul>
<li><p class="no-margin"><b>No cachees indefinidamente</b>: la lista de empresas accesibles
cambia cuando el propietario invita/expulsa al usuario, cuando se
crean empresas nuevas o cuando el usuario se da de baja. Refresca
al iniciar sesión y cuando una llamada a otra empresa devuelva
<code>401</code>/<code>403</code>.</p></li>
<li><p class="no-margin"><b><code>role</code> informa, no autoriza</b>: el rol en una empresa no
determina por sí solo qué endpoints puedes llamar. La autorización
efectiva la dan los <b>scopes</b> del access token. Para descubrir
qué scopes tienes, decodifica el JWT o consulta la
<a href="https://help.facturadirecta.com/es/articles/13538825">guía de Autenticación</a>.</p></li>
<li><p class="no-margin"><b>Para API Keys, no llames a <code>/profile</code></b>: la API key ya conoce su
empresa. Si tu integración alterna entre OAuth y API Key,
detéctalo en el cliente y omite el paso.</p></li>
</ul>
<p class="no-margin"></p>
<h2 id="errores-comunes"><b>Errores comunes</b></h2>
<p class="no-margin"></p>
<ul>
<li><p class="no-margin"><code>401 Unauthorized</code> (<code>code: &quot;auth_*&quot;</code>) — el <code>access_token</code> no es
válido o ha expirado. Refresca el token vía OAuth.</p></li>
<li><p class="no-margin"><code>403 Forbidden</code> — intento de llamar con una API key. Este endpoint
es OAuth-only por diseño.</p></li>
</ul>
<p class="no-margin"></p>
