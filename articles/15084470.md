<h1 id="perfil-de-usuario">Perfil de usuario</h1><p class="no-margin"></p>
<p class="no-margin">El recurso <code>profile</code> devuelve los datos básicos del <b>usuario autenticado</b> y la lista de <b>empresas a las que tiene acceso</b>. Es la operación que usa toda integración OAuth para resolver qué <code>companyId</code> puede usar en el resto de llamadas.</p><p class="no-margin"></p>
<h2 id="excepciones-de-este-endpoint">Excepciones de este endpoint</h2><p class="no-margin"></p>
<p class="no-margin"><code>/profile</code> es el único endpoint de la API pública que rompe con dos convenciones del resto del catálogo:</p><p class="no-margin"></p>
<ul><li><p class="no-margin"><b>No lleva <code>companyId</code> en la ruta.</b> El path es literalmente <code>/profile</code>: el usuario puede tener acceso a varias empresas, y este endpoint sirve precisamente para descubrirlas antes de fijar una.</p></li><li><p class="no-margin"><b>Es OAuth-only.</b> Las <a href="https://help.facturadirecta.com/es/articles/13538824" target="_blank" class="intercom-content-link">API Keys</a> <b>no pueden llamar</b> a este endpoint: una API key está ya ligada a una empresa concreta, así que no tiene sentido pedir su perfil de empresas.</p></li></ul><p class="no-margin"></p>
<h2 id="scope">Scope</h2><p class="no-margin"></p>
<p class="no-margin">El scope efectivo es <code>profile</code>. No aparece como permiso seleccionable porque <b>se incluye automáticamente</b> en el cliente OAuth <code>facturadirecta-api</code> al hacer login. No hace falta pedirlo explícitamente.</p><p class="no-margin"></p>
<h2 id="estructura">Estructura</h2><p class="no-margin"></p>
<p class="no-margin">El endpoint devuelve <code>{ profile }</code> con la siguiente forma:</p><p class="no-margin"></p>
<div class="intercom-interblocks-table-container"><table role="presentation"><tbody><tr><td><p class="no-margin">Campo</p></td><td><p class="no-margin">Tipo</p></td><td><p class="no-margin">Significado</p></td></tr><tr><td><p class="no-margin"><code>username</code></p></td><td><p class="no-margin">string</p></td><td><p class="no-margin">Email del usuario autenticado (es el identificador único).</p></td></tr><tr><td><p class="no-margin"><code>email</code></p></td><td><p class="no-margin">string</p></td><td><p class="no-margin">Mismo valor que <code>username</code>. Se mantiene por compatibilidad.</p></td></tr><tr><td><p class="no-margin"><code>firstName</code></p></td><td><p class="no-margin">string</p></td><td><p class="no-margin">Nombre del usuario (del campo <code>given_name</code> del JWT).</p></td></tr><tr><td><p class="no-margin"><code>lastName</code></p></td><td><p class="no-margin">string</p></td><td><p class="no-margin">Apellidos del usuario (del campo <code>family_name</code> del JWT).</p></td></tr><tr><td><p class="no-margin"><code>companies</code></p></td><td><p class="no-margin">array</p></td><td><p class="no-margin">Empresas a las que tiene acceso este usuario.</p></td></tr></tbody></table></div><p class="no-margin"></p>
<p class="no-margin">Cada elemento de <code>companies</code>:</p><p class="no-margin"></p>
<div class="intercom-interblocks-table-container"><table role="presentation"><tbody><tr><td><p class="no-margin">Campo</p></td><td><p class="no-margin">Tipo</p></td><td><p class="no-margin">Significado</p></td></tr><tr><td><p class="no-margin"><code>id</code></p></td><td><p class="no-margin">string</p></td><td><p class="no-margin">ID de empresa. Es el valor que debes usar como <code>{companyId}</code> en las llamadas al resto de la API.</p></td></tr><tr><td><p class="no-margin"><code>name</code></p></td><td><p class="no-margin">string</p></td><td><p class="no-margin">Nombre de la empresa. Si la empresa tiene primer apellido (<code>surname</code>), aparece concatenado con un espacio.</p></td></tr><tr><td><p class="no-margin"><code>taxCode</code></p></td><td><p class="no-margin">string</p></td><td><p class="no-margin">CIF/NIF de la empresa.</p></td></tr><tr><td><p class="no-margin"><code>brand</code></p></td><td><p class="no-margin">string</p></td><td><p class="no-margin">Marca comercial, si está informada (opcional).</p></td></tr><tr><td><p class="no-margin"><code>owner</code></p></td><td><p class="no-margin">string</p></td><td><p class="no-margin">Email del propietario de la empresa.</p></td></tr><tr><td><p class="no-margin"><code>role</code></p></td><td><p class="no-margin">string</p></td><td><p class="no-margin">Rol del usuario autenticado en esa empresa concreta.</p></td></tr></tbody></table></div><p class="no-margin"></p>

<ul><li><p class="no-margin"><a href="#obtener-el-perfil" target="_blank" class="intercom-content-link">Obtener el perfil</a></p></li></ul><p class="no-margin"></p>
<h2 id="obtener-el-perfil">Obtener el perfil</h2><p class="no-margin"></p>
<p class="no-margin"><code>GET /profile</code> devuelve el perfil del usuario autenticado y sus empresas.</p><p class="no-margin"></p>
<p class="no-margin"><b>No lleva parámetros de path ni de consulta.</b> Solo el header <code>accept-version</code> para fijar la versión de la API. Ver <a href="https://help.facturadirecta.com/es/articles/13538825" target="_blank" class="intercom-content-link">Autenticación</a>.</p><p class="no-margin"></p>
<h3 id="copy-as-curl">Copy as cURL</h3><p class="no-margin"></p>
<pre><code>curl -s -H "Authorization: Bearer $ACCESS_TOKEN" \  "https://app.facturadirecta.com/api/profile"</code></pre><p class="no-margin"></p>
<p class="no-margin">Respuesta típica:</p><p class="no-margin"></p>
<pre><code>{  "profile": {    "username": "ana@ejemplo.com",    "email": "ana@ejemplo.com",    "firstName": "Ana",    "lastName": "García López",    "companies": [      {        "id": "com_3a7e8d29-4f5b-4c6e-9a1d-2b8c4e7f5d3a",        "name": "Estudio Diseño SL",        "taxCode": "B12345674",        "owner": "ana@ejemplo.com",        "role": "admin"      },      {        "id": "com_8b1c5e94-2d6a-4f7e-9c3b-1a5d8f4e2c7b",        "name": "Cooperativa Norte",        "taxCode": "F87654328",        "brand": "Norte",        "owner": "luis@cooperativa.com",        "role": "accountant"      }    ]  }}</code></pre><p class="no-margin"></p>
<h2 id="flujo-tpico-de-integracin">Flujo típico de integración</h2><p class="no-margin"></p>
<ol><li><p class="no-margin">El usuario completa el login OAuth. Tu integración recibe el <code>access_token</code>.</p></li><li><p class="no-margin">Llamas a <code>GET /profile</code> para descubrir las empresas accesibles.</p></li><li><p class="no-margin">Eliges una empresa (UI: dejas que el usuario seleccione; o automatización: filtras por <code>taxCode</code> o <code>id</code> conocido).</p></li><li><p class="no-margin">Usas su <code>id</code> como <code>{companyId}</code> en todas las llamadas siguientes a la API.</p></li></ol><p class="no-margin"></p>
<h2 id="recomendaciones">Recomendaciones</h2><p class="no-margin"></p>
<ul><li><p class="no-margin"><b>No cachees indefinidamente</b>: la lista de empresas accesibles cambia cuando el propietario invita/expulsa al usuario, cuando se crean empresas nuevas o cuando el usuario se da de baja. Refresca al iniciar sesión y cuando una llamada a otra empresa devuelva <code>401</code>/<code>403</code>.</p></li><li><p class="no-margin"><b><code>role</code> informa, no autoriza</b>: el rol en una empresa no determina por sí solo qué endpoints puedes llamar. La autorización efectiva la dan los <b>scopes</b> del access token. Para descubrir qué scopes tienes, decodifica el JWT o consulta la <a href="https://help.facturadirecta.com/es/articles/13538825" target="_blank" class="intercom-content-link">guía de Autenticación</a>.</p></li><li><p class="no-margin"><b>Para API Keys, no llames a <code>/profile</code></b>: la API key ya conoce su empresa. Si tu integración alterna entre OAuth y API Key, detéctalo en el cliente y omite el paso.</p></li></ul><p class="no-margin"></p>
<h2 id="errores-comunes">Errores comunes</h2><p class="no-margin"></p>
<ul><li><p class="no-margin"><code>401 Unauthorized</code> (<code>code: "auth_*"</code>) — el <code>access_token</code> no es válido o ha expirado. Refresca el token vía OAuth.</p></li><li><p class="no-margin"><code>403 Forbidden</code> — intento de llamar con una API key. Este endpoint es OAuth-only por diseño.</p></li></ul><p class="no-margin"></p>
