<p class="no-margin">El recurso <code>invoices</code> te permite gestionar las facturas de venta de tu empresa. Puedes crear facturas, consultarlas, actualizarlas, eliminarlas, registrar pagos y enviarlas por email.</p><p class="no-margin"></p>
<p class="no-margin">Este artículo incluye ejemplos prácticos para los casos de uso más comunes: facturas con distintos tipos de IVA, con retención IRPF, con suplidos, con precios IVA incluido y con pagos.</p><p class="no-margin"></p>
<p class="no-margin">💡 <b>Recomendación</b>: Antes de empezar a desarrollar, te recomendamos <a href="https://help.facturadirecta.com/es/articles/12608263-donde-puedo-crear-un-entorno-de-pruebas-sandbox-para-la-api" target="_blank" class="intercom-content-link">crear un entorno de pruebas (sandbox)</a> para hacer tus primeras llamadas sin afectar tus datos reales.</p><p class="no-margin"></p>

facturas para diferentes casos de uso. Todos los ejemplos usan el método POST a <code>/{companyId}/invoices</code>.</p><p class="no-margin"></p>
<h2 id="h_ejemplo_minimal">Factura básica (minimal) </h2><p class="no-margin">El caso más simple: una factura con una sola línea y un tipo de IVA.</p><p class="no-margin"></p>
<pre><code>curl -X POST \  -H "facturadirecta-api-key: TU_CLAVE_API" \  -H "Content-Type: application/json" \  -d '{    "content": {      "type": "invoice",      "main": {        "docNumber": {          "series": "F"        },        "contact": "con_10000000-0000-4000-8000-000000000000",        "currency": "EUR",        "lines": [          {            "quantity": 1,            "unitPrice": 100,            "tax": ["S_IVA_21"],            "text": "Descripción del servicio"          }        ]      }    }  }' \  https://app.facturadirecta.com/api/TU_COMPANY_ID/invoices</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Resultado:</b> Una factura de 100€ + 21% IVA = 121€ total.</p><p class="no-margin"></p>
<h2 id="h_ejemplo_taxes">Factura con varios tipos de IVA</h2><p class="no-margin">Cuando vendes productos con distintos tipos impositivos (21%, 10%, 4%).</p><p class="no-margin"></p>
<pre><code>{  "content": {    "type": "invoice",    "main": {      "docNumber": {        "series": "F"      },      "contact": "con_10000000-0000-4000-8000-000000000000",      "currency": "EUR",      "lines": [        {          "quantity": 1,          "unitPrice": 100,          "tax": ["S_IVA_21"],          "text": "Producto IVA 21%"        },        {          "quantity": 1,          "unitPrice": 50,          "tax": ["S_IVA_10"],          "text": "Producto IVA 10%"        },        {          "quantity": 1,          "unitPrice": 20,          "tax": ["S_IVA_4"],          "text": "Producto IVA 4%"        }      ]    }  }}</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Resultado:</b></p><ul><li><p class="no-margin">Línea 1: 100€ + 21€ IVA</p></li><li><p class="no-margin">Línea 2: 50€ + 5€ IVA</p></li><li><p class="no-margin">Línea 3: 20€ + 0,80€ IVA</p></li><li><p class="no-margin"><b>Total:</b> 196,80€</p></li></ul><h2 id="h_ejemplo_irpf">Factura con retención IRPF</h2><p class="no-margin">Para profesionales que aplican retención en sus facturas.</p><p class="no-margin"></p>
<pre><code>{  "content": {    "type": "invoice",    "main": {      "docNumber": {        "series": "F"      },      "contact": "con_10000000-0000-4000-8000-000000000000",      "currency": "EUR",      "lines": [        {          "quantity": 1,          "unitPrice": 100,          "tax": ["S_IVA_21", "S_IRPF_15"],          "text": "Servicios profesionales"        }      ]    }  }}</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Resultado:</b></p><ul><li><p class="no-margin">Base: 100€</p></li><li><p class="no-margin">IVA 21%: +21€</p></li><li><p class="no-margin">IRPF 15%: -15€</p></li><li><p class="no-margin"><b>Total a pagar:</b> 106€</p></li></ul><h2 id="h_ejemplo_accounting">Factura con cuentas contables personalizadas </h2><p class="no-margin">Cuando necesitas contabilizar líneas en cuentas específicas del plan contable.</p><p class="no-margin"></p>
<pre><code>{  "content": {    "type": "invoice",    "main": {      "docNumber": {        "series": "F"      },      "contact": "con_10000000-0000-4000-8000-000000000000",      "currency": "EUR",      "lines": [        {          "quantity": 1,          "unitPrice": 100,          "tax": ["S_IVA_21"],          "text": "Producto estándar",          "account": "700000"        },        {          "quantity": 1,          "unitPrice": 250,          "tax": ["S_IVA_21"],          "text": "Producto especial contabilizado en la cuenta 700001",          "account": "700001"        }      ]    }  }}</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Nota:</b> El campo <code>account</code> indica la cuenta contable donde se registrará el ingreso de esa línea.</p><p class="no-margin"></p>
<h2 id="h_ejemplo_suplidos">Factura con suplidos</h2><p class="no-margin">Los suplidos son gastos que has pagado en nombre de tu cliente y que debes repercutir sin IVA.</p><p class="no-margin"></p>
<pre><code>{  "content": {    "type": "invoice",    "main": {      "docNumber": {        "series": "F"      },      "contact": "con_10000000-0000-4000-8000-000000000000",      "currency": "EUR",      "lines": [        {          "quantity": 1,          "unitPrice": 100,          "tax": ["S_IVA_21"],          "text": "Servicios profesionales"        },        {          "quantity": 1,          "unitPrice": 50,          "tax": ["S_SUPLIDOS"],          "text": "Suplidos Registro Mercantil",          "document": "pro_suplido",          "account": "566000"        }      ]    }  }}</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Resultado:</b></p><ul><li><p class="no-margin">Servicios: 100€ + 21€ IVA = 121€</p></li><li><p class="no-margin">Suplidos: 50€ (sin IVA)</p></li><li><p class="no-margin"><b>Total:</b> 171€ </p></li></ul><h2 id="h_ejemplo_taxincluded">Factura con precios IVA incluido</h2><p class="no-margin">Útil cuando trabajas con precios finales (PVP) y necesitas que FacturaDirecta calcule la base imponible.</p><p class="no-margin"></p>
<pre><code>{  "content": {    "type": "invoice",    "main": {      "docNumber": {        "series": "F"      },      "contact": "con_10000000-0000-4000-8000-000000000000",      "taxIncludedPrices": true,      "currency": "EUR",      "lines": [        {          "quantity": 2,          "unitPrice": 100,          "tax": ["S_IVA_21"],          "text": "Producto PVP 100€"        }      ]    }  }}</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Resultado:</b></p><ul><li><p class="no-margin">Total línea: 2 x 100€ = 200€ (IVA incluido)</p></li><li><p class="no-margin">Base calculada: 165,29€</p></li><li><p class="no-margin">IVA 21%: 34,71€</p></li><li><p class="no-margin"><b>Total:</b> 200€</p></li></ul><h2 id="h_ejemplo_paid">Factura pagada automáticamente</h2><p class="no-margin">Crea una factura y registra el pago completo en un solo paso.</p><p class="no-margin"></p>
<pre><code>{  "content": {    "type": "invoice",    "main": {      "docNumber": {        "series": "F"      },      "contact": "con_10000000-0000-4000-8000-000000000000",      "currency": "EUR",      "lines": [        {          "quantity": 1,          "unitPrice": 100,          "tax": ["S_IVA_21"],          "text": "Descripción del servicio"        }      ]    }  },  "payments": [    {      "bank": "ban_10000000-0000-4000-8000-000000000000"    }  ]}</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Nota:</b> Al indicar solo el <code>bank</code> sin <code>amount</code>, FacturaDirecta entiende que el pago es por el importe total de la factura.</p><p class="no-margin"></p>
<h2 id="h_ejemplo_multiplepayments">Factura con múltiples pagos </h2><p class="no-margin">Cuando el cliente ha pagado en varios plazos antes de emitir la factura.</p><p class="no-margin"></p>
<pre><code>{  "content": {    "type": "invoice",    "main": {      "docNumber": {        "series": "F"      },      "date": "2026-05-01",      "contact": "con_10000000-0000-4000-8000-000000000000",      "currency": "EUR",      "lines": [        {          "quantity": 1,          "unitPrice": 100,          "tax": ["S_IVA_21"],          "text": "Descripción del servicio"        }      ]    }  },  "payments": [    {      "date": "2026-04-15",      "bank": "ban_10000000-0000-4000-8000-000000000000",      "amount": 50    },    {      "date": "2026-04-25",      "bank": "ban_10000000-0000-4000-8000-000000000000",      "amount": 50    },    {      "bank": "ban_10000000-0000-4000-8000-000000000000"    }  ]}</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Resultado:</b> Factura de 121€ con tres pagos:</p><ul><li><p class="no-margin">15/04: 50€</p></li><li><p class="no-margin">25/04: 50€</p></li><li><p class="no-margin">01/05: 21€ (resto pendiente)</p></li></ul><p class="no-margin"></p>
<h2 id="h_4a4862ecb7">Factura rectificativa</h2><p class="no-margin">Para emitir una factura rectificativa vinculada a la factura original, indica el ID de esa factura en el campo <code>correctedInvoice</code>.</p><p class="no-margin"></p>
<pre><code>{<br>  "content":<br>  {<br>    "type": "invoice",<br>    "main":<br>    {<br>      "docNumber":<br>      {<br>        "series": "R"<br>      },<br>      "contact": "con_10000000-0000-4000-8000-000000000000",<br>      "correctedInvoice": "inv_10000000-0000-4000-8000-000000000000",<br>      "currency": "EUR",<br>      "lines":<br>      [<br>        {<br>          "quantity": 1,<br>          "unitPrice": -100,<br>          "tax":<br>          [<br>            "S_IVA_21"<br>          ],<br>          "text": "Rectificación de la factura A-2026-0001"<br>        }<br>      ],<br>      "facturae":<br>      {<br>        "Corrective_ReasonCode": "01",<br>        "Corrective_CorrectionMethod": "02",<br>        "Corrective_AdditionalReasonDescription": "Error en el importe facturado"<br>      },<br>      "ticketbai":<br>      {<br>        "claveTipoFacturaRectificativa": "R1"<br>      }<br>    }<br>  }<br>}</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Notas:</b></p><ul><li><p class="no-margin">El campo <code>correctedInvoice</code> debe apuntar a una factura existente del mismo cliente (contact).</p></li><li><p class="no-margin">Para que los importes resten respecto a la factura original, se suelen usar valores negativos en <code>quantity</code> o <code>unitPrice</code>.</p></li><li><p class="no-margin">Los campos <code>main.facturae.*</code> solo son necesarios si la empresa emite Facturae:</p><ul><li><p class="no-margin"><code>Corrective_ReasonCode</code>: motivo. Códigos "01"–"16" (errores RD 1496/2003) y "80"–"85" (Art. 80 Ley 37/92 IVA).</p></li><li><p class="no-margin"><code>Corrective_CorrectionMethod</code>: "02" (por diferencias) o "03" (descuento por volumen).</p></li><li><p class="no-margin"><code>Corrective_AdditionalReasonDescription</code>: descripción adicional opcional (máx. 2.500 caracteres).</p></li><li><p class="no-margin">El campo <code>main.ticketbai.claveTipoFacturaRectificativa</code> solo es necesario si la empresa emite TicketBAI. Valores: "R1", "R2", "R3", "R4", "R5".</p></li></ul></li></ul><p class="no-margin"></p>

<h1 id="h_estados">Estados de una factura</h1><p class="no-margin">Las facturas pueden tener los siguientes estados:</p><div class="intercom-interblocks-table-container"><table role="presentation"><tbody><tr><td><p class="no-margin"><b>Estado</b></p></td><td><p class="no-margin"><b>Descripción</b></p></td></tr><tr><td><p class="no-margin">draft</p></td><td><p class="no-margin">Borrador: factura en edición, sin numerar</p></td></tr><tr><td><p class="no-margin">pending</p></td><td><p class="no-margin">Pendiente de pago: tiene saldo pendiente pero no ha vencido</p></td></tr><tr><td><p class="no-margin">overdue</p></td><td><p class="no-margin">Vencida: el saldo pendiente ha superado la fecha de vencimiento</p></td></tr><tr><td><p class="no-margin">paid</p></td><td><p class="no-margin">Pagada: saldo pendiente es cero</p></td></tr><tr><td><p class="no-margin">overpaid</p></td><td><p class="no-margin">Sobrepagada: los pagos superan el importe de la factura</p></td></tr><tr><td><p class="no-margin">voided</p></td><td><p class="no-margin">Anulada: factura cancelada</p></td></tr></tbody></table></div><h1 id="h_errores">Errores comunes</h1><h2 id="h_error_contact">Error: "contact is required" </h2><p class="no-margin">El campo <code>contact</code> es obligatorio. Asegúrate de incluir el ID de un cliente existente.</p><p class="no-margin"></p>
<h2 id="h_error_lines">Error: "lines must have at least 1 item"</h2><p class="no-margin">Una factura necesita al menos una línea. Verifica que el array <code>lines</code> no está vacío.</p><p class="no-margin"></p>
<h2 id="h_error_tax">Error: "tax is required for each line"</h2><p class="no-margin">Cada línea debe incluir el campo <code>tax</code> con al menos un código de impuesto. Si la línea está exenta, usa el código de exención correspondiente.</p><p class="no-margin"></p>
<h2 id="h_error_series">Error: "series not found" </h2><p class="no-margin">La serie indicada en <code>docNumber.series</code> no existe en la configuración de tu empresa. Verifica las series disponibles en Ajustes &gt; Series de documentos.</p><p class="no-margin"></p>
<h2 id="h_error_bank">Error: "bank not found"</h2><p class="no-margin">El ID de banco indicado en los pagos no existe. Consulta las cuentas bancarias disponibles con <code>GET /{companyId}/banks</code>.</p>
