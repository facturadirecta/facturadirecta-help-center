<p class="no-margin">El recurso <code>contacts</code> te permite gestionar los contactos de tu empresa: clientes, proveedores y empleados. Cada contacto puede tener uno o varios roles según la relación comercial que tengas con él.</p><p class="no-margin"></p>
<p class="no-margin">Este artículo incluye ejemplos prácticos para los casos más comunes: clientes españoles (empresa y particular), clientes intracomunitarios (con validación VIES) y clientes extracomunitarios.</p><p class="no-margin"></p>
<p class="no-margin">💡 <b>Recomendación</b>: Antes de empezar a desarrollar, te recomendamos <a href="https://help.facturadirecta.com/es/articles/12608263-donde-puedo-crear-un-entorno-de-pruebas-sandbox-para-la-api" target="_blank" class="intercom-content-link">crear un entorno de pruebas (sandbox)</a> para hacer tus primeras llamadas sin afectar tus datos reales.</p><p class="no-margin"></p>
<h1 id="h_endpoints">Endpoints disponibles</h1><div class="intercom-interblocks-table-container"><table role="presentation"><tbody><tr><td style="width: 129px;"><p class="no-margin"><b>Método</b></p></td><td style="width: 270px;"><p class="no-margin"><b>Endpoint</b></p></td><td><p class="no-margin"><b>Descripción</b></p></td></tr><tr><td style="width: 129px;"><p class="no-margin">GET</p></td><td style="width: 270px;"><p class="no-margin">/{companyId}/contacts</p></td><td><p class="no-margin">Lista de contactos (con filtros y paginación)</p></td></tr><tr><td style="width: 129px;"><p class="no-margin">POST</p></td><td style="width: 270px;"><p class="no-margin">/{companyId}/contacts</p></td><td><p class="no-margin">Crear un contacto</p></td></tr><tr><td style="width: 129px;"><p class="no-margin">GET</p></td><td style="width: 270px;"><p class="no-margin">/{companyId}/contacts/{id}</p></td><td><p class="no-margin">Obtener un contacto por ID</p></td></tr><tr><td style="width: 129px;"><p class="no-margin">PUT</p></td><td style="width: 270px;"><p class="no-margin">/{companyId}/contacts/{id}</p></td><td><p class="no-margin">Actualizar un contacto</p></td></tr><tr><td style="width: 129px;"><p class="no-margin">DELETE</p></td><td style="width: 270px;"><p class="no-margin">/{companyId}/contacts/{id}</p></td><td><p class="no-margin">Eliminar un contacto</p></td></tr><tr><td style="width: 129px;"><p class="no-margin">PUT</p></td><td style="width: 270px;"><p class="no-margin">/{companyId}/contacts/{id}/tags</p></td><td><p class="no-margin">Actualizar etiquetas del contacto</p></td></tr></tbody></table></div><h1 id="h_ejemplos">Ejemplos de código</h1><p class="no-margin">Los siguientes ejemplos muestran cómo crear contactos para diferentes casos de uso. Todos los ejemplos usan el método POST a <code>/{companyId}/contacts</code>.</p><p class="no-margin"></p>
<h2 id="h_ejemplo_spain_company">Cliente español (empresa) </h2><p class="no-margin">El caso más común: una empresa española identificada por su CIF.</p><p class="no-margin"></p>
<pre><code>curl -X POST \  -H "facturadirecta-api-key: TU_CLAVE_API" \  -H "Content-Type: application/json" \  -d '{    "content": {      "type": "contact",      "main": {        "name": "Cliente Empresa SL",        "fiscalId": "B12345674",        "currency": "EUR",        "country": "ES",        "address": "Pza Mayor, 4",        "zipcode": "49004",        "city": "Zamora",        "accounts": {          "client": "430000",          "clientCredit": "438000"        }      }    }  }' \  https://app.facturadirecta.com/api/TU_COMPANY_ID/contacts</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Nota:</b> FacturaDirecta valida automáticamente el formato del CIF español y asigna la posición fiscal correspondiente.</p><p class="no-margin"></p>
<h2 id="h_ejemplo_spain_individual">Cliente español (particular)</h2><p class="no-margin">Para personas físicas con DNI/NIE. Usa el campo <code>surname</code> para los apellidos y marca <code>endCustomer: true</code>.</p><p class="no-margin"></p>
<pre><code>{  "content": {    "type": "contact",    "main": {      "name": "Pedro",      "surname": "García Gómez",      "fiscalId": "82931482Z",      "currency": "EUR",      "country": "ES",      "address": "Av. de Leopoldo Alas Clarín, 2",      "zipcode": "49018",      "city": "Zamora",      "endCustomer": true,      "accounts": {        "client": "430000",        "clientCredit": "438000"      }    }  }}</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Importante:</b> El campo <code>endCustomer: true</code> indica que es un consumidor final. Esto afecta a cómo se genera el asiento contable y a algunos informes fiscales.</p><p class="no-margin"></p>
<h2 id="h_ejemplo_eu">Cliente intracomunitario (UE) </h2><p class="no-margin">Para clientes de otros países de la Unión Europea. Usa <code>fiscalId</code> con <code>fiscalIdType: "02"</code> (NIF-IVA intracomunitario) e incluye la validación VIES.</p><p class="no-margin"></p>
<pre><code>{  "content": {    "type": "contact",    "main": {      "name": "Google Ireland Limited",      "fiscalId": "IE6388047V",      "fiscalIdType": "02",      "vatEUValidation": {        "valid": true,        "date": "2026-01-15T10:30:00+01:00"      },      "currency": "EUR",      "country": "IE",      "address": "Gordon House, Barrow Street",      "city": "Dublin 4",      "accounts": {        "client": "430000",        "clientCredit": "438000"      }    }  }}</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Campos clave:</b></p><ul><li><p class="no-margin"><code>fiscalId</code>: NIF intracomunitario con prefijo del país (ej: IE para Irlanda)</p></li><li><p class="no-margin"><code>fiscalIdType: "02"</code>: Indica que es un NIF-IVA intracomunitario</p></li><li><p class="no-margin"><code>vatEUValidation.valid</code>: Resultado de la validación en VIES</p></li><li><p class="no-margin"><code>vatEUValidation.date</code>: Fecha en que se realizó la validación</p></li></ul><p class="no-margin"><b>Nota:</b> La validación VIES es obligatoria para aplicar la exención de IVA en operaciones intracomunitarias. Si no tienes la validación, incluye <code>vatEUValidation.valid: false</code>.</p><p class="no-margin"></p>
<h2 id="h_ejemplo_other">Cliente extracomunitario</h2><p class="no-margin">Para clientes fuera de la Unión Europea. Puedes incluir el identificador fiscal del país de origen usando <code>fiscalIdType: "06"</code> (otro documento probatorio).</p><p class="no-margin"></p>
<pre><code>{  "content": {    "type": "contact",    "main": {      "name": "Mailgun Technologies",      "fiscalId": "12-3456789",      "fiscalIdType": "06",      "currency": "USD",      "country": "US",      "address": "535 Mission St., 14th Floor",      "zipcode": "94105",      "city": "San Francisco",      "region": "CA",      "accounts": {        "client": "430000",        "clientCredit": "438000"      }    }  }}</code></pre><p class="no-margin"></p>
<p class="no-margin"><b>Nota:</b> Se recomienda usar los campos estructurados (<code>address</code>, <code>zipcode</code>, <code>city</code>, <code>region</code>) para la dirección de todos los contactos, incluyendo extranjeros.</p><p class="no-margin"></p>

<h1 id="h_errores">Errores comunes</h1><h2 id="h_error_fiscalid">Error: "Invalid fiscalId format"</h2><p class="no-margin">El CIF/NIF/NIE no tiene un formato válido. Verifica:</p><ul><li><p class="no-margin">CIF de empresa: letra + 8 dígitos (ej: B12345674)</p></li><li><p class="no-margin">NIF persona física: 8 dígitos + letra (ej: 82931482Z)</p></li><li><p class="no-margin">NIE: X/Y/Z + 7 dígitos + letra (ej: Y1234567X)</p></li></ul><h2 id="h_error_vateu">Error: "vatEU must include country prefix"</h2><p class="no-margin">El NIF intracomunitario debe incluir el prefijo del país. Ejemplo: <code>IE6388047V</code> (no solo <code>6388047V</code>).</p><p class="no-margin"></p>
<h2 id="h_error_country">Error: "Invalid country code"</h2><p class="no-margin">Usa códigos de país ISO 3166-1 Alpha-2 de dos letras: ES, FR, DE, IT, US, etc.</p><p class="no-margin"></p>
<h2 id="h_error_name">Error: "name is required"</h2><p class="no-margin">El campo <code>name</code> es obligatorio. Para personas físicas, indica el nombre de pila y usa <code>surname</code> para los apellidos.</p><p class="no-margin"></p>
